# 实验计划

## 研究问题

Lavish + Treehouse + No Mistakes 是否能实质改善个人 AI builder 的 `Plan -> Code -> Validate` 工作流？

## 实验

1. 仓库侦察：记录元数据、入口、安装说明、依赖和许可证。
2. 安装与 smoke test：尽量运行官方或最小安全命令。
3. 核心能力测试：Lavish 规划 artifact、Treehouse worktree、No Mistakes review/validation。
4. 集成工作流：构建 Talking Breads V1 静态网站。
5. 基线比较：Markdown spec + raw git worktree + 手动验证。
6. 边界失败测试：缺 auth/API key/命名冲突/无效 repo/失败测试等。
