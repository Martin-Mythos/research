Start by creating a new folder for your work with a short, descriptive kebab-case name.

Create a `notes.md` file in that folder and append notes as you work. Track:

- the original prompt or research question
- commands, scripts, data sources, and URLs actually used
- experiments attempted and their outcomes
- failed paths and why they were abandoned
- verification evidence, including test commands and outputs worth preserving

Build a `README.md` report at the end of the investigation. The report should be understandable without reading the transcript and should clearly separate verified findings from speculation.

Final commits should include only the research folder and selected artifacts from it:

- `notes.md` and `README.md`
- code, scripts, benchmark harnesses, generated charts, and small data files created for the investigation
- diffs against external repositories saved as patch files, if external code was modified
- selected binary artifacts only when they are essential and smaller than 2 MB

Do not commit full copies of repositories, vendor directories, downloaded dependency trees, credentials, private data, or raw external datasets unless the dataset is explicitly redistributable and necessary for reproducibility.

Do not create `_summary.md` manually. Summary files are generated by automation after commits land on `main`.

Prefer reproducible experiments over narrative-only answers. If a claim can be tested with code, write and run the code.
