# Experiment Plan

## Research question

Can a last30days-style, evidence-scored, recency-oriented workflow provide more traceable and actionable findings than a one-off desktop research sweep for GLM-5.2 engineering reality and the Anthropic Mythos/Fable 5 export-control directive?

## Track 1: Traditional Baseline Sweep

1. Search current web sources for GLM-5.2 launch, benchmarks, context window, pricing/quota, and engineering caveats.
2. Search current web sources for Anthropic Mythos/Fable 5, the U.S. government directive, foreign-national access scope, customer impact, and compliance implications.
3. Save source snapshots and extracted claims into `artifacts/baseline_raw.json`.
4. Produce a static narrative summary.

Expected weakness: a baseline sweep can summarize what is currently published, but it tends to flatten source types and can over-weight launch-day marketing or headline-level policy framing.

## Track 2: 30Days-Skill Incremental Framework

The 30-day pattern is compressed into 30 research intervals. Each interval has a vector, evidence requirement, metric slot, contradiction check, and confidence score.

### GLM-5.2 interval vectors

- **Intervals 1-10**: benchmark and instruction-drift audit. Compare public scores against evaluation settings, tool access, context length, and anti-cheating controls.
- **Intervals 11-20**: latency and serving audit. Track quota multipliers, peak/off-peak constraints, provider claims, KV-cache pressure, concurrency, and CPU-side scheduling overhead.
- **Intervals 21-30**: long-context stress audit. Track needle-in-haystack reliability, repo-scale context retention, reward-hacking guardrails, and failure modes under messy coding-agent trajectories.

### Anthropic Mythos/Fable 5 interval vectors

- **Intervals 1-10**: directive scope verification. Track first-party statement, affected models, affected user classes, foreign-national definition, and unaffected model list.
- **Intervals 11-20**: supply-chain and compliance implementation. Track customer interruption, employee access constraints, identity segmentation, tenant controls, and legal/compliance analysis.
- **Intervals 21-30**: strategic resilience. Track sovereign-AI response, open-weight alternatives, provider fallback patterns, policy rollback/clarification, and future AI security rubric signals.

## Enhanced modules included

1. **Leaderboard Deflation Test**: GLM-5.2 claims are scored against evaluation coverage, harness assumptions, and known long-context serving bottlenecks.
2. **Tech Resilience Index**: Mythos/Fable 5 is evaluated through provider centralization, access-segmentation feasibility, customer fallback capacity, open-weight substitutes, and sovereign-AI pressure.
3. **KOL Insight Generator**: the final report includes short security/AI industry insight drafts comparing baseline and incremental outputs.

## Safety constraints

No paid API keys, no credential use, no production model abuse, no network attacks, and no fabricated benchmark execution. Missing data is recorded as unknown.
