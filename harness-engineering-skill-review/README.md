## 6. 全面能力剖析：`yao-tutorial-skill`

### 6.1 优势
1. **流程化强**：输入归一化→研究→大纲→写作→配图→导出→验证，闭环完整。
2. **交付导向强**：不仅产文，还强调多格式产物和可验证性。
3. **对新手友好**：明确要求“把概念拆成问题/例子/步骤”，降低阅读门槛。
4. **证据意识强**：将内部证据映射与公开成品分离，兼顾可读性与审计性。

### 6.2 局限
1. **执行成本高于普通写作**：完整流程要求更多时间与工具链（截图、导出器、验证器）。
2. **对主题深度依赖资料质量**：若输入资料薄弱，虽可外检索，但质量上限受限。
3. **“教程化”偏好明显**：不适合超短答复、纯观点文或仅视觉输出任务。

### 6.3 风险点
1. **伪引用风险**：若研究取证环节偷工减料，可能出现“结构漂亮但证据不足”。
2. **格式完成度与内容完成度错配**：多格式导出齐全不代表知识正确。
3. **图文一致性风险**：章节图若自动生成但未人工校对，可能与正文逻辑偏离。

### 6.4 适配建议
- 作为企业内部“知识资产生产线”时，建议新增：
  1. 事实核查门禁（抽样逐条回源）。
  2. 术语一致性检查（glossary lint）。
  3. 教程可运行示例仓（最小代码项目 + CI）联动。

`yao-tutorial-skill` 在“把复杂主题系统化教学交付”方面表现优秀，尤其适合需要**结构化、可审计、可复用**教程生产的团队。
