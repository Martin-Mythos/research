# Sense Deck Skill 与主流 AI PPT 工具链的架构实证与横评报告

## 1. 实验摘要 (Executive Summary)
本次实验聚焦 `xwbcl123/-PPT-sense-deck-skill-`（仓库内名称“鲸格PPT”）的模板优先工作流：先构建内容 IR，再从 full-deck / single-page layout / theme 中装配，最后输出可运行 HTML。实证结果显示：

- **Sense Deck 的核心优势**是“前端可控性高 + 视觉表达上限高 + 可脱离特定渲染器运行”。
- **归藏（Marp）流派优势**是“结构化快、写作与版本管理友好”，但在精细排版和复杂视觉上受限于 Markdown 语法壳。
- **花叔（Python-pptx）流派优势**是“企业办公链路兼容高（PPTX 可直接进 Office）”，但工程门槛高，且反复调版成本显著。

总体结论：若目标是“浏览器原生演示 + 高可定制 + 团队可二次开发”，Sense Deck 更适合作为中台基座；若目标是“快速文档化演示”，Marp 仍是低摩擦方案；若目标是“强 Office 交付”，Python-pptx 更稳。

## 3. HTML PPT 交付件与架构解析 (Artifacts & Architecture)

### 技术点评
- **分屏逻辑**：采用 `section.slide` 作为最小幻灯片单元，通过 `active` 类切换显隐；`show(n)` 负责循环翻页。
- **信息密度控制**：每页仅 1 个主结论 + 2~4 个支撑点，避免“文字墙”。
- **排版策略**：`grid-2` + `card` 做稳定的信息容器，覆盖议程、风险矩阵、路线图等通用页型。
- **交互路径**：支持 `ArrowRight/Left`、`PageDown/Up`、点击翻页，满足现场演示基本需求。

## 4. 评估基准与横评矩阵 (Rubrics & Comparison)

### 4.1 量化 Rubrics（1-5 分）
1. **DOM 结构与渲染路径（Engineering）**：可编辑性、可维护性、脱离模型后的改稿成本
2. **视觉扩展与信息噪声比（Visual & Layout）**：留白控制、图文混排、视觉一致性、跨设备适配
3. **逻辑穿透力（Logic & Sense）**：自动拆页质量、重点提炼、叙事推进
4. **工具链依赖风险**：是否强依赖特定运行时、插件、桌面软件
5. **团队协作友好度**：版本控制、多人协作、审阅与回溯成本

### 4.3 深度点评

#### A. 前端可控性
- **Sense Deck**：直接操控 DOM/CSS/JS，能做细粒度动画、响应式和组件化；适合前端工程团队。
- **归藏（Marp）**：以 Markdown 驱动，表达效率高，但复杂布局常需自定义 HTML/CSS 插入。
- **花叔（Python-pptx）**：可精准控制 PPT 对象，但“代码改样式”迭代慢，视觉实验成本较高。

#### B. 内容转化率（提纲 → 可演示成品）
- **Sense Deck**：在模板与布局资产丰富时，能快速产出“即演示”成品。
- **归藏**：文本到初稿最快，适合讲稿导向场景；视觉精修阶段可能进入“样式补丁期”。
- **花叔**：结构化强，适合标准化企业模板批量出片，但第一次打磨成本高。

#### C. 工具链依赖（依赖依赖）
- **Sense Deck**：主要依赖浏览器，部署轻；但非技术用户面对 HTML/CSS 仍有门槛。
- **归藏**：依赖 Markdown + Marp，学习曲线最平缓。
- **花叔**：依赖 Python 环境与 pptx 生态，且最终审稿通常仍回到 Office。

## 5. 落地建议 (Recommendations)
1. **Sense Deck 技术局限（客观）**
   - 对非代码用户不够友好：HTML/CSS 微调存在技能门槛。
   - 若缺少模板资产库，项目初期会出现“工程化过重”。
   - 浏览器渲染在不同设备字体与色彩上可能有细微差异，需要验收基线。

2. **面向研发/安全团队的选型建议**
   - 若你们已有前端工程能力，建议以 **Sense Deck（HTML）为主引擎**，并沉淀组织级布局资产与主题系统。
   - 若你们强调“会议纪要/方案说明快速产出”，可采用 **归藏（Marp）作为快写入口**，再转 Sense Deck 做关键场合精修。
   - 若你们必须以 Office 为最终交付，保留 **Python-pptx 支线**，用于批量化/模板化出件。
   - 最佳实践是“三层流水线”：
     - **L1 内容层**：统一提纲 IR（标题、结论、证据、备注）；
     - **L2 渲染层**：Marp 快速稿 + Sense Deck 演示稿 + pptx 交付稿；
     - **L3 治理层**：版本审计、合规校验、指标看板。
