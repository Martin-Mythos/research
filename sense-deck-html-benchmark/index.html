02 议程

逃逸边界 = 可控域 与 不可控域 的动态交界

可控域（治理对象）

  • 系统提示与模板
  • RAG 检索白名单
  • 工具调用策略
  • 输出过滤规则

不可控域（风险来源）

  • 用户恶意输入
  • 第三方数据污染
  • 隐式指令劫持
  • 模型推理幻觉
03 边界定义

五类高危逃逸路径与业务后果

提示注入

覆盖上层策略，诱导泄露系统指令。

越权工具调用

Agent 访问敏感 API，触发数据破坏。

机密外泄

上下文拼接失误导致 PII/密钥暴露。

模型串扰

多 Agent 信息污染，错误决策放大。

04 风险图谱

防御机制：五层联动，不依赖单点能力

L1

输入层
语义风控+注入检测

L2

上下文层
最小化检索+脱敏

L3

执行层
工具权限沙箱

L4

输出层
合规过滤+事实核验

L5

审计层：全链路日志、异常回放、责任归因。

目标：≥85%

目标：≤30%

响应闭环时长（P1）

目标：≤4 小时

06 指标闭环

落地路线图：30/60/90 天

0-30 天

  • 梳理资产与数据分级

31-60 天
